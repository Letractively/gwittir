115     //This method creates a Huffman encoding table as a
116     // HashMap object that relates the variable length
117     // bitcodes to the characters in the original message.
118     // The bitcodes are constructed as objects of type
119     // StringBuffer consisting of sequences of the characters
120     // 1 and 0.
121     //Each bitcode describes the traversal path from the root
122     // of the Huffman tree to a leaf on that tree.  Each time
123     // the path turns to the left, a 0 character is appended
124     // onto the StringBuffer object and becomes part of the
125     // resulting bitcode.  Each time the path turns to the
126     // right, a 1 character is appended onto the
127     // StringBuffer object.  When a leaf is reached, the
128     // character stored in that leaf is retrieved and put
129     // into the HashMap object as a key.  A String
130     // representation of the StringBuffer object is used as
131     // the value for that key in the HashMap.
132     //At completion,the HashMap object contains a series of
133     // keys consisting of the original characters in the
134     // message and a series of values as String objects
135     // (consisting only of 1 and 0 characters) representing
136     // the bitcodes that will eventually be used to encode
137     // the original message.
138     //Note that theHashMap object that is populated by this
139     // method is the data structure that is required later
140     // to decode the encoded message.
141     void createBitCodes(HuffTree tree) {
142         if (tree instanceof HuffNode) {
143             // This is a node, not a leaf.  Process it as a node.
144 
145             //Cast to type HuffNode.
146             HuffNode node = (HuffNode) tree;
147 
148             // Get and save the left and right branches
149             HuffTree left = node.getLeft();
150             HuffTree right = node.getRight();
151 
152             //Append a 0 onto the StringBuffer object.  Then make
153             // a recursive call to this method passing a
154             // reference to the left child as a parameter.  This
155             // recursive call will work its way all the way down
156             // to a leaf before returning.  Then it will be time
157             // to process the right path.
158             code.append("0");
159             createBitCodes(left);
160 
161             //Return to here from recursive call on left child.
162 
163             //Delete the 0 from the end of the StringBuffer
164             // object to restore the contents of that object to
165             // the same state that it had before appending the 0
166             // and making the recursive call on the left branch.
167             //Now we will make a right turn.  Append a 1 to the
168             // StringBuffer object and make a recursive call to
169             // this method passing a reference to the right child
170             // as a parameter.  Once again, this recursive call
171             // will work its way all the  way down to a leaf
172             // before returning.
173             code.deleteCharAt(code.length() - 1); //Delete the 0.
174             code.append("1");
175             createBitCodes(right);
176 
177             //Return to here from recursive call on right child.
178 
179             //Delete the character most recently appended to the
180             // StringBuffer object and return from this recursive
181             // call to the method.  The character is deleted
182             // because control is being transferred back one
183             // level in the recursive process and the
184             // StringBuffer object must be restored to the same
185             // state that it had when this recursive call was
186             // made.
187             code.deleteCharAt(code.length() - 1);
188         } else {
189             //This is a leaf.  Process it as such.
190             //Cast the object to type HuffLeaf.
191             HuffLeaf leaf = (HuffLeaf) tree;
192 
193             //Put an entry into the HashMap.  The HashMap
194             // key consists of the character value stored in the
195             // leaf. The value in the HashMap consists of the
196             // contents of the StringBuffer object representing
197             // the path from the root of the tree to the leaf.
198             // This is the bitcode and is stored in the HashMap
199             // as a String consisting of only 1 and 0 characters.
200             huffEncodeTable.put((char) (leaf.getValue()), code.toString());
201         } //end else
202     } //end createBitCodes

234     // consisting of properly linked HuffNode objects and
235     // HuffLeaf objects.

242     // Huffman tree (as represented by the single HuffTree
243     // object).
244     //The method contains two sections of code that can be
245     // enabled to display:
246     // 1. The contents of the original TreeSet object.
247     // 2. The contents of the TreeSet object for each
248     //    iteration during which HuffLeaf objects are being
249     //    combined with HuffNode objects to form the final
250     //    HuffTree object.

252     // Huffman tree is constructed.  However, this code
253     // should be enabled only for small trees because it

256     //The HuffTree object is constructed by:
257     // 1. Extracting pairs of HuffLeaf or HuffNode objects
258     //    from the TreeSet object in ascending order based
259     //    on their frequency value.
260     // 2. Using the pair of extracted objects to construct
261     //    a new HuffNode object where the two extracted
262     //    objects become children of the new HuffNode
263     //    object, and where the frequency value stored in
264     //    the new HuffNode object is the sum of the
265     //    frequency values in the two child objects.
266     // 3. Removing the two original HuffLeaf or HuffNode
267     //    objects from the TreeSet and adding the new
268     //    HuffNode object to the TreeSet object.  The
269     //    position of the new HuffNode object in the Treeset
270     //    object is determined by its frequency value
271     //    relative to the other HuffNode or HuffLeaf objects
272     //    in the collection. The new HuffNode object will
273     //    eventually become a child of another new HuffNode
274     //    object unless it ends up as the root of the
275     //    HuffTree object.
276     // 4. Continuing this process until the TreeSet object
277     //    contains a single object of type HuffTree.
278     void createHuffTree() {
279         //Enable the following statements to see the original
280         // contents of the TreeSet object. Do this only for
281         // small trees because it generates lots of output.

293         //Iterate on the size of the TreeSet object until all
294         // of the elements have been combined into a single

296         while (theTree.size() > 1) {
297             //Get, save, and remove the first two elements.
298             HuffTree left = theTree.first();

322                   }//end while loop
323                   //End code to display the TreeSet

325         } //end while

437     //This method encodes an incoming String message using
438     // the Huffman encoding methodology.  The method also
439     // receives a reference to an empty data structure.
440     // This data structures is populated with encoding
441     // particulars required later by the decode method
442     // to decode and transform the encoded message back
443     // into the original String message.  Note that in
444     // order to keep this method simple, pad characters may

571     //The purpose of this method is to create actual bit data
572     // that matches the 1 and 0 characters in the
573     // stringEncodedData that represents bits with the 1 and
574     // 0 characters.
575     //Note that this method doesn't handle the end of the
576     // data very gracefully for those cases where the number
577     // of required bits is not a multiple of 8.  It simply
578     // adds enough "0" characters to the end to cause the
579     // length to be a multiple of 8.  This may result in
580     // extraneous characters at the end of the decoded
581     // message later. However, it isn't difficult to remove
582     // the extraneous characters at decode time as long as
583     // the length of the original message is known.

586     // for comparison with the non-extended version.
587     //Note that the binary Huffman-encoded data produced by
588     // this method is stored in a data structure of type
589     // ArrayList <Byte>.
590     void encodeStringToBits() {
591         //Extend the length of the stringEncodedData to cause
592         // it to be a multiple of 8.
593         int remainder = stringEncodedData.length() % 8;
594 
595         for (int cnt = 0; cnt < (8 - remainder); cnt++) {
596             stringEncodedData += "0";
597         } //end for loop
598 
599         //For illustration purposes only, enable the following
600         // two statements to display the extended
601         // stringEncodedData in the same format as the
602         // original stringEncodedData.
603 
604         //Extract the String representations of the required
605         // eight bits.  Generate eight actual matching bits by
606         // looking the bit combination up in a table.
607         for (int cnt = 0; cnt < stringEncodedData.length(); cnt += 8) {
608             String strBits = stringEncodedData.substring(cnt, cnt + 8);
609             byte realBits = encodingBitMap.get(strBits);
610             binaryEncodedData.add(realBits);
611         }
612     }
